# Design audit — Day 5 (current) + Day 4 (historical)

## Day 5 reversal

The Day 4 Italiana + Fraunces direction was **rejected**. The user's
verbatim correction:

> "It still reads generic as fuck to me, we def need to make this more
> unique. I think the main thing is that I hate serif fonts on websites,
> it looks like placeholder shit no matter the site imo."

Hard rule going forward: **NO serif fonts on websites.** Italiana,
Fraunces, DM Sans, Inter, Geist, JetBrains Mono — all out. The Day 5
type system collapses to two non-serif families:

- **Space Grotesk** (variable) drives display, body, large numerics,
  and the wordmark. Tailwind `font-sans` maps here.
- **Space Mono** drives captions, labels, bracketed metadata, code-like
  signals. Tailwind `font-mono` maps here.

The Day 5 direction is **brutalist + arctic blue**:

- Palette: stone-950 base + amber-200 primary + arctic blue `#4FA3FF`
  secondary (`colors.arctic.{200,400,500,700}`) for signals / focused
  states / info / hover-on-interactive. Emerald stays for healthy /
  progress.
- Heavy 2-3px borders on cards / buttons / inputs. Default
  `rounded-none`; `rounded-sm` caps at 2px.
- Flat opaque block cards. NO translucent gradients, NO backdrop blur,
  NO editorial shadow.
- All-caps display headings via the `.display-xl` utility (Space
  Grotesk 700, uppercase, tracking -0.03em, clamped 2.5rem-4rem).
- Mono captions inside `[ BRACKETS ]` via the `.label-mono` utility
  (Space Mono 10px, uppercase, tracking 0.08em). Brackets are part of
  the rendering, not the copy.
- Ornament marks `◆ ▸ → ↗ █` as decoration / list bullets / transition
  glyphs. Shipped via the new `Ornament.tsx` primitive.
- Stats render LARGE — primary tiles at `text-6xl md:text-7xl`,
  tabular numerics, slashed zero via `font-feature-settings "zero"`.
- Sidebar active item is a 3px amber left bar + uppercase Space Mono
  label. No background fill.
- Per-route headers use the new `RouteHeader` primitive: section
  number `[ §0N / NAME ]` + uppercase display title + Space Mono
  subhead with `↗` arrow ornament + heavy bottom rule.

### What was reverted from Day 4

| Day 4 pattern | Day 5 replacement |
|---|---|
| Italiana wordmark | Space Grotesk 700 uppercase tracking-tight |
| Fraunces display / body / mono | Space Grotesk (display + body) + Space Mono (captions) |
| `.caption-label` (Fraunces uppercase weight 500) | `.label-mono` (Space Mono uppercase + brackets) |
| Layered translucent cards (`bg-gradient + ring + shadow-editorial + backdrop-blur`) | Flat opaque cards with `border-2 border-stone-700`, optional 3px accent bar |
| Underline-animated amber focus (`.focus-underline`) | Arctic-blue 3px focus underline (`.focus-arctic`) — and on the secondary button family the focus ring is arctic + 2px offset |
| Soft-tint pills (`bg-X/30 + ring-1`) | Border-only pills with bracket glyphs inside |
| Hover-lift + amber gradient on primary buttons | Flat amber primary block + active translate-down |
| Toast slide-in with rotation | Toast slide-in from the right, no rotation, 3px top tone bar |
| Modal 1px gradient amber accent | Modal 3px solid amber top border |
| Paper-grain noise overlay + warm radial gradients | Flat stone-950 background |
| Editorial italic body copy | Mono body copy with bracket conventions |

### Day 5 mock-api → real api-client swap

Workstream B carried in alongside the visual reset:

- `/lib/api-client.ts` — one typed function per `API_ROUTES` entry.
  Each call validates the response through a Zod schema composed from
  `/contracts/models.ts` (or pulled directly from `/contracts/api.ts`
  where the response schema is already exported). 4xx/5xx responses
  flow through `ApiErrorBodySchema` and throw a typed `ApiClientError`
  with the server's `code` for switch-statement narrowing.
- BYOK forwarding: AI routes take an explicit `apiKey` argument. The
  TanStack Query hook layer pulls the plaintext from the new
  `useByokKey` in-memory store via `readByokKeyOrThrow()`. Settings's
  Save and Unlock flows populate the store. The plaintext never enters
  localStorage and never persists across tab close.
- Every `/lib/queries/*` hook now imports from `@/lib/api-client`.
  `/lib/mock-api.ts` stays in the repo (Day 6 deletion target) but
  has zero remaining importers — `rg "from '@/lib/mock-api'" --type ts`
  returns zero hits outside the file's own header comment.

# Design audit — Day 4 (historical)

The Day 3 refresh (Italiana wordmark + Fraunces display + paper grain) was
typography-only. Layout, spacing, colour, micro-interactions, and form
chrome stayed at the Tailwind / shadcn default. The result still reads as
"generic dark-app starter generated by an LLM."

This file audits every shared component and every route for the specific
class patterns that signal "default", and lists the changes Day 4 is
landing. It is a working document for this PR, not a long-lived spec.

---

## Hard rules carried in from the Day 4 kickoff

1. **Two font families total.** Italiana for the wordmark; Fraunces for
   everything else (body, display, captions, numerics). No DM Sans, no
   JetBrains Mono, no Inter, no system default.
2. **Layout / spacing / colour / motion must move**, not just typography.
3. No new dependency without a one-line ARCHITECTURE.md justification.
4. Stone-950 + amber-200 palette stays; we extend it with a secondary
   emerald accent for "healthy / progress" signals and warmer sepias for
   marginalia. No third *family* of hues.

---

## Shared component tells and fixes

### Card.tsx
**Today:** every Card renders as `bg-stone-950/60 border border-stone-800/60 rounded-md backdrop-blur-[2px]`. Identical across every route, every section, every weight of content. Padding is `p-4` / `p-5` / `p-10`, no asymmetric variation.

**Fix:** layered translucent surface — `bg-gradient-to-b from-stone-900/40 via-stone-950/50 to-stone-950/60`, `ring-1 ring-stone-100/5` instead of a solid border, soft inset shadow. Optional `accent` prop adds a 2px amber left bar for "this card matters more." Optional `tone="urgent"` swaps the ring to rose-900/40 and inflects the gradient. The base card stays cheap; the variants are opt-in.

### Button.tsx
**Today:** four flat variants, no rest-state dimming, no hover lift, no press depth. `primary` is the only one that visually pops.

**Fix:** rest state dims secondary to 80% opacity, hover lifts the button by `translate-y-[-1px]` with a shadow growth (`shadow-amber-200/0` to `shadow-amber-200/10`), active state presses back down. Primary picks up a subtle vertical gradient so it doesn't read as a single flat amber slab.

### Input.tsx / Textarea.tsx
**Today:** boxed dark form fields with `bg-stone-900/80 border border-stone-800 rounded-sm`. Same chrome as every shadcn-derived app on the internet.

**Fix:** underline-style. Transparent background, `border-b border-stone-700`, label above in Fraunces small-caps with wide tracking. Focus replaces the border-b with a 2px amber-200 line that animates in from the left over 150ms. A subtle filled fallback variant stays for the BYOK password field (where masked-text needs a frame to read).

### Field.tsx
**Today:** `text-[10px] uppercase tracking-[0.2em] text-stone-500 font-mono` on every label. Reads as mono-caption-spam.

**Fix:** Fraunces uppercase, `tracking-[0.18em]`, weight 500. Hint text uses Fraunces italic at smaller size. Required indicator is a single amber bullet, not an asterisk.

### Modal.tsx
**Today:** plain dark panel, generic backdrop, X-only close. Header is `bg-stone-950 border-b border-stone-800/80`.

**Fix:** 1px amber top accent on the modal frame, backdrop blur bumped from 4px to 8px with a slight warm tint. Close button transitions colour over 100ms instead of stepping. Entry animation: 200ms fade + 6px slide-up. Already keyboard-trap and ESC-aware; that stays.

### Sidebar.tsx
**Today:** active item is a stone-900 fill with amber text. Looks like every nav menu ever shipped.

**Fix:** active item has a 2px amber bar pinned to the left edge of the row + transparent background, with the label in amber and slightly wider tracking. Hover state widens letter-spacing 0.02em (subtle but feels intentional). Vertical breathing increased from `space-y-0.5` to `space-y-1`. Footer adds a quiet mood line ("Two hours, applications only") that the user can interpret as a daily nudge.

### Toaster.tsx
**Today:** background-coloured toast tiles in the bottom-right. No entry motion.

**Fix:** toasts slide in from the right with a small rotation (0.5deg) for handcraft feel, hold for 5s, slide out. Top accent bar (2px) instead of full background tint; the body stays stone-950 so the chrome stays editorial. Success / error / info differentiated by the top bar colour.

## Per-route tells and fixes

### /dashboard
**Today:** four uniform stat cards in a `grid-cols-2 md:grid-cols-4 gap-3`. Recent applications list is a `divide-y` with a subtle hover. Three didactic-paragraph cards at the bottom feel like a chrome banner.

**Fix:** stat tiles get weight differentiation — "Interviews" and "Response %" are large and accented; "Applied" and "In flight" are quieter. Recent-applications rows get a 3px coloured left-bar keyed to status (interview = emerald, applied = amber, screen = stone). Dashboard hero copy moves to a single line with the date as a subhead in Fraunces italic. The "compound effect" trio gets cut from the Day 4 surface; it's documentation pretending to be UI.

### /skills
**Today:** profile card on top, then a vertical stack of identical job cards.

**Fix:** profile card adopts a two-column layout on desktop — name + positioning on the left, contact + target-roles on the right, with an asymmetric divider in between. Job cards get a hairline left-bar in the wordmark amber + slightly larger title (Fraunces 2xl). Projects inside a job sit in an offset block with a warm stone-900 background — clearly a sub-section, not a list continuation.

### /discovery
**Today:** filter buttons render identically whether active or inactive; posting cards are uniform 3-column-y blocks.

**Fix:** filters dim inactive buttons to `text-stone-600` so the active one carries the eye. Posting cards get a right-side 4px alignment-score bar (emerald ≥ 85, amber ≥ 70, stone < 70). Top posting (highest alignmentScore) renders as a *featured* card — taller, fuller description, primary CTA visible without hovering. Subsequent cards render tight.

### /tracker
**Today:** uniform application rows with status pill at the right edge.

**Fix:** status moves to a left-anchored coloured bar (3px) keyed to stage. Alignment score grows from `text-sm w-12` to `text-lg tabular-nums` with a subtle amber-tinted background pill — it's the primary metric, treat it that way. The filter tabs collapse to a single horizontal row with the active filter underlined rather than boxed.

### Application detail drawer
**Today:** status chip grid, alignment card, follow-up + notes, JD details summary, events timeline.

**Fix:** alignment card becomes the focal point — bigger score, radial warm gradient behind it, recommendation in Fraunces italic, missing-keywords pills carry a subtle rose-900 ring. Timeline gains a `border-l` rail with a coloured dot at each event (status_change = amber, note = stone, document_generated = emerald). Status chips get a check icon on the active one.

### /documents
**Today:** generation panel + flat document list + preview modal.

**Fix:** generation panel becomes a single-row composition — kind pill + application selector + Generate CTA — with the kind icon (FileText, Mail, etc.) sized large enough to read at glance. List rows pick up the kind colour as a left-bar. Preview modal grows a small toolbar above the body (Copy, Open in new tab) so it's an actionable surface.

### /interviews
**Today:** Mock and Stories tabs identical in styling; transcript reads as a list of small labelled spans.

**Fix:** Mock and Stories use different visual languages — Mock is a vertical chat with interviewer bubbles left-aligned and user replies right-aligned; Stories is a magazine-style article spread with the role title in display and each STAR section labelled with a small uppercase Fraunces caption. Tab switching uses a thin underline that slides between the two.

### /settings
**Today:** form blocks in the standard editorial palette.

**Fix:** API key card splits into "key" + "passphrase + strength" sub-sections with explicit visual separation. Passphrase strength meter gets a labelled segment animation: bands fill left-to-right as strength rises. "Skip passphrase" toggle gets rose-tinted treatment (it's a downgrade choice).
